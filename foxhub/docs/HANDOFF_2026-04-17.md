## Recovery Notes

- If the landing page does not appear, check `src/App.jsx` route selection before changing auth behavior.
- Do not remove `/signin`; it is intentionally separate from the public landing page.
- If mobile layout breaks, start with the phone media queries and overflow guards in `src/styles.css`.
- If mobile wrappers look stale, rerun `npm run sync:ios` and `npm run sync:android`, then check the asset names in each wrapper's `public/index.html`.
- For Android native builds, use Java 21: `JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64 ./gradlew assembleDebug`.
- For iOS native builds, use a machine with Xcode and signing configured.
- If Firebase persistence rejects a new state write, review `firestore.rules` before changing store state shape.
- If Growth OS disappears or stops saving, check `growthCategories`, `growthEvents`, and `growthOutputs` in `src/useFoxHubStore.js`, `src/repository-firebase.js`, and `firestore.rules`.
- If Growth OS grouping looks wrong, check `GROWTH_OS_GROUPS`, `growthCategory()`, and `GROWTH_OS_CATEGORIES` in `src/useFoxHubStore.js`.
- If operator access stops working for legitimate staff, set a Firebase Auth custom claim of `platformOperator: true` for that user's UID from a trusted server/admin tool.
- If ops webhooks must be used, set `FOXHUB_OPS_WEBHOOK_SECRET` and deploy Functions after the Firebase project supports Functions deployment.
- If blue accents reappear after future edits, check `src/styles.css` for `--accent`, `--accent-soft`, old blue hex values, and old blue `rgba(...)` values, then rebuild, redeploy, and sync the mobile wrappers.
- If the old `F` logo reappears, check the landing brand in `src/App.jsx`, the header and nav glyph rendering in `src/FoxHubShell.jsx`, and the shared SVG in `src/FoxHeadMark.jsx`.
- If Start Near Me stops saving city, check the `updateProfile` prop passed from `src/App.jsx` into `src/FoxHubShell.jsx` and the existing profile city field before adding a new datastore shape.
