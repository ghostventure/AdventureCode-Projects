# FoxHub OneID Rollout - 2026-04-19

FoxHub now presents the account system as one shared OneID across the app.

## What Changed

- Added `oneId` to the profile model and self user-record snapshots.
- OneID is generated as a short `FOX-...` marker from the Firebase UID when available, with local fallback generation from account/profile identifiers.
- Local fallback profiles now preserve or generate a OneID.
- Firebase profiles now persist OneID in `users/{uid}`.
- Firestore rules now allow and validate the `oneId` field.
- The sign-in/onboarding page and profile panel now explain and display the OneID account marker.
- Web Hosting and Firestore rules were deployed after the change.
- iOS and Android Capacitor wrappers were synced with the updated web bundle.

## Recovery Notes

- OneID is generated by `buildOneId()` and resolved by `resolveProfileOneId()` in `src/identity.js`.
- Account profiles are still stored in `users/{uid}` for Firebase mode and browser storage for local mode.
- If Firestore rejects profile writes after OneID edits, check `validOneId()`, `validProfileBase()`, and `validProfileUpdate()` in `firestore.rules`.
- After account/profile edits, rerun `npm test`, `npm run build`, `npm run sync:android`, and `npm run sync:ios`.
- Because this rollout changes Firestore rules, deploy Hosting and Firestore rules together.
