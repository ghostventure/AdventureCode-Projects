## Recovery Notes

- Rebuild `mobile-web` after changing root `.html`, `.css`, `.js`, or `.webmanifest` files:
  - `npm run build:mobile`
- Run deploys from the CLTCH repo root only.
- Do not re-enable website service-worker caching for auth/dashboard paths without re-testing sign-in recovery. It was intentionally disabled after stale cached code caused auth instability.
- If phone layouts regress, start in `cltch-boilerplate.css`, then rebuild `mobile-web`.
- If the Loaded Kit disappears, check `getCltchComponentRegistry()` in `site-init.js`, then rerun `npm run build:mobile`.
- If the Dispatch Kit becomes one endless list again, check `#cltchComponentList` in `cltch-boilerplate.css`.
- If the Android APK needs to be rebuilt, use Java 21: `JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64 ./gradlew assembleDebug`.
