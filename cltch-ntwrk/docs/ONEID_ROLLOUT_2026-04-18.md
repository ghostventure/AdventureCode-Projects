# CLTCH OneID Rollout - 2026-04-18

CLTCH.NTWRK now presents the account system as one shared OneID across the network.

## What Changed

- Added `oneId` to the shared user summary builder.
- OneID is generated from the Firebase account UID as a short `CLTCH-...` account marker.
- Firestore user-summary rules now allow the `oneId` field.
- The sign-in page now explains that one account works across hosting, performing, bookings, radar, support, and payments.
- Support diagnostics now displays the OneID.
- Host and performer profile pages now show the OneID beside account activity.
- Terms and Privacy copy now mention the shared OneID account model in plain wording.

## Recovery Notes

- OneID is generated by `buildOneId()` in `app/user-db.js`.
- Account summaries are still stored in `users/{uid}`.
- Role routing still uses `userRoles/{uid}` plus the summary role.
- After account-helper or page edits, rerun `npm run build:mobile`.
- Because this rollout changes Firestore rules, deploy hosting and Firestore rules together.
