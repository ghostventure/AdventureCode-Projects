## Functional mechanics

Every component now supports:

- `Enable`
  - Persists the component in `localStorage` under `cltch_component_installs_v1`.
  - Records enabled time, route, name, and category.
- `Run`
  - Enables the component if needed.
  - Writes a per-component action record under `cltch_component_action_v1:{componentId}`.
  - Adds a recent activity entry.
  - Shows a user-facing toast.
- `Open`
  - Routes the user to the most relevant existing CLTCH page or anchor.

## Implementation note

This is the appropriate static-site install pattern for CLTCH.NTWRK. The components are now actionable and routable across the current site. Components that require real payment processors, messaging backends, identity vendors, or external notification systems still need backend/provider activation before they can become full production subsystems.
