Less guessing after the match.

Once a request turns into a booking, CLTCH keeps the basic operating details in one place so both sides can move with fewer loose messages.

One booking room for the work. Hosts and performers should be able to see the request, contact status, arrival timing, event notes, payout path, and review step without hunting through separate threads.

  • Role, rate, date, location, arrival window, and event expectations.

  • Payment setup, completion notes, dispute trail, and review handoff.
  • Plain-language account paths for host setup and performer setup.
